Abora Features

This is primarily a collection of future features for Abora that is culled from my understanding of Xanadu, Udanax-Gold and other pieces on hypertext systems.

The intent of this list is to both help with an overview of what the final implementati.on of Abora may grow into, and also to act as an indication of the current location on this road.

. = Not Started
- = Started but not visible in the release
I = Initial implementation
O = Reasonable implementation
X = Finished

  • Links
    • Fine Grained [I] - links can be specified down to individual character or position
    • Typed [I] - for example Author, Refute,
    • 2 Way [I] - find all pages that link to you
    • External to source document [.] - anyone can create a link to/from a document that do not own assuming it is visible
    • Overlapping [I] - no limitations on the number of links to from the same content, unlike html where a single character can only be in the source of a single link
    • Links to links [.] - in addition to linking to content, links can also be connected to existing links
    • Non-contiguous [I] - source of a link could be the five most important important sentences of a document, even if they occurred thoughout the document.
    • Filtering [I] - user filtering of links to focus attention and hide chaff
    • Version survivability [I] - links continue to point to any of the original text that exists in later versions
    • Complex media [.] - in addition to text contents, links can be made to image pixels and other spaces
  • Transclusion - or virtual copies of content
    • Core Transclusion [I] -
    • Differences [O] - Find and show similarity between content
  • WWW compatability
    • Outgoing links [I] - Links to external URLs from an Abora document
    • Read-Only webserver [.] - integration with a webserver supplying read-only HTML translations of Abora pages
    • In-Browser editing [.] - Access full Abora functionality from within an WWW browser, by either plugin or heroic scripting
  • Infrastructure
    • Client-Server [I] - Access a single server from one or more client applications
    • Server-Server [.] - Distributed network of servers so support client access to more data than held by the immediately accessible server
    • Client API [.] - Cross platform and language independent client API for communication with the server
    • Secure comms [.] - SSL like support for communication between machines
    • Payment Scheme [.] - Support secure payment scheme
    • Sponsored Docs [.] - Multiple locations can sponsor a document to ensure it isn't lost
    • Garbage Collection [.] - Documents and sub-parts are only removed from the system if they aren't being used
    • Sensors [.] - Learn when a new edition of a work is made, or a new link is made to an edition.
